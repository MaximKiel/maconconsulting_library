Некоторые комментарии по тонкостям работы поиска проектов:

1. Для лучшего понимания логики поиска и хранения данных в БД рекомендуется сперва прочитать инструкцию по внесению
проектов в БД библиотеки.

2. На странице поиска приводятся критерии поля для заполнения - критерии поиска. Возможные критерии практически
совпадают с полями проекта.

3. Поиск возможен по произвольному числу критериев. Если не ввести ни одного критерия для поиска, в результате будут
показаны все проекты, внесенные в БД библиотеки.

4. Поле критериев, по которым поиск не нужен, следует оставить пустыми. Исключение - поле "Год проекта". В этом поле
по-умолчанию стоит ноль. Если нет необходимости вести поиск по году, нужно оставить ноль в поле ввода.

5. В каждом проекте есть поля, которые предусматривают возможность ввода нескольких значений. Это поля: страны, регионы,
города (МО).
    Например: "Россия, Армения, Казахстан"
Однако при заполнении критериев поиска можно задать только одно значение.
    Например: "Россия" или "Армения"
Поиск найдет и выдаст в результате все проекты, в которых в поле было указано искомое значение, даже если этих значений
изначально было введено несколько.
    Например: условный проект "23100Р_Маркетинг_Россия, Армения, Казахстан" будет выводиться при запросе в поиске любой
    из указанных стран - "Россия" или "Армения" или "Казахстан".

6. При заполнении строковых полей поиска (все поля, кроме года), не имеет значения регистр букв.
    Например: в поиске в поле "Страна" можно ввести значение "россия". Даже в таком случае в результате будут выведены
    все проекты, в которых в соответствующем поле было указано "Россия".
    В то же время, при заполнении изначальных полей проекта при его создании рекомендуется соблюдать корректный регистр
    букв - на поиск это не повлияет, но сделает отображение проектов на сайте визуально красивее.

После заполнения всех необходимых критериев необходимо нажать кнопку "Найти" внизу страницы!
