Некоторые комментарии по тонкостям работы поиска проектов:

1. Для лучшего понимания логики поиска и хранения данных рекомендуется сперва прочитать инструкцию по внесению
проектов в библиотеку.

2. На странице поиска приводятся критерии поля для заполнения - критерии поиска. Возможные критерии практически
совпадают с полями проекта.

3. Поиск возможен по произвольному числу критериев. Если не ввести ни одного критерия для поиска, в результате будут
показаны все проекты, внесенные в библиотеку.

4. Поле критериев, по которым поиск не нужен, следует оставить пустыми. Исключение - поле "Год проекта". В этом поле
по умолчанию стоит ноль. Если нет необходимости вести поиск по году, нужно оставить ноль в поле ввода.

5. В каждом проекте есть поля, которые предусматривают возможность ввода строки из нескольких значений.
Это поля: название, заказчик, актуальность данных, локация. Поиск осуществляется по совпадению элемента строки, поэтому искать
можно даже по части названия.
    Например: если в проекте указан заказчик "СберРешения", проект можно будет найти по запросу в соответствующем поле
              для поиска строки "Сбер".

6. При заполнении строковых полей поиска не имеет значения регистр букв. В то же время, при заполнении изначальных
полей проекта при его создании рекомендуется соблюдать корректный регистр букв - на поиск это не повлияет, но сделает
отображение проектов на сайте визуально приятнее.

7. Несмотря на то, что при создании проекта можно выбрать несколько сегментов рынка, форматов отчета и ключевых слов,
для поиска можно выбрать только один элемент по каждому из параметров.

После заполнения всех необходимых критериев необходимо нажать кнопку "Найти" внизу страницы!
