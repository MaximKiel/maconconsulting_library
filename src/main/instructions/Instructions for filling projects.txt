При создании нового проекта заполняются следующие поля:

1. Номер проекта (поле обязательное, уникальное) в формате цифр (без "Р" на конце).
    Например: "23001", "23002"
Для квартальных отчетов особый порядок заполнения: сначала цифровой код, затем квартал, затем город.
    Например: "23000_1Q_Краснодар", "23000_3Q_Москва"

2. Год проекта (поле обязательное) в формате 4 цифр.
    Например: "2023", "2022"

3. Актуальность данных в формате "месяц.год".
    Например: "12.2023"

4. Название проекта (поле обязательное, уникальное) в текстовом формате. Название полностью повторяет название
проектной папки на сервере.
    Например: "23003Р. Ялта. МИ МЖС. Knight Frank", "23001Р. Ташкент. МИ. МЖС. CMWP"

5. Юрлицо и/или другое обозначение клиента (поле обязательное) в текстовом формате. Важно! Название организации записывать
без организационно-правовой формы и без кавычек!
    Например: Интеркомп
Если, помимо официального названия существует общепринятое, его следует записывать через запятую в этом же поле.
Аналогично следует поступать, если заказчиков несколько.
    Например: Интеркомп, СберРешения

6. Страны, в которых проводилось исследование (поле обязательное) в текстовом формате. Если проект включает несколько стран,
записывать их нужно, отделяя друг от друга запятой и пробелом (необходимо для последующей корректной работы поиска).
    Например: "Россия, Армения, Казахстан"

7. Регионы, в которых проводилось исследование в текстовом формате. Рекомендуется записывать полное официальное название
региона, а не его сокращенный вариант, для унификации данных. Если проект включает несколько регионов, записывать их нужно,
отделяя друг от друга запятой и пробелом (необходимо для последующей корректной работы поиска).
    Например: "Краснодарский край, Республика Татарстан, Республика Северная Осетия - Алания"

8. Название городов (муниципальных образований), в которых проводилось исследование в текстовом формате.
Рекомендуется записывать полное официальное название города (муниципального образования) для унификации данных.
Если проект включает несколько городов, записывать их нужно, отделяя друг от друга запятой и пробелом (необходимо
для последующей корректной работы поиска).
    Например: "Сочи, Анапа, Геленджик"
Если исследование проводилось в небольшом населенном пункте, который является частью другого муниципального образования,
необходимо записать оба наименования через запятую (в таком случае проект будет найден и по названию населенного пункта,
и по названию его муниципального образования).
    Например: "Ольгинка, Туапсинский район", "Кабардинка, Геленджик"
ВАЖНО: перед названием населенного пункта НЕ ставится обозначение его типа: город, село, пгт и др.
    Например: НЕПРАВИЛЬНО - "с. Дивноморское, г-к. Геленджик"
              ПРАВИЛЬНО - "Дивноморское, Геленджик"

9. Сегменты рынка, по которым проводилось исследование (поле обязательное). Можно выбрать одно или несколько значений
из перечня (чтобы выбрать несколько, зажмите клавишу Control/Command). Сам перечень можно редактировать, если перейти
по ссылке под полем.

10. Тип отчета (поле обязательное). Можно выбрать единственное значение из перечня. Сам перечень можно редактировать,
если перейти по ссылке под полем.

11. Форматы отчета (поле обязательное). Можно выбрать одно или несколько значений из перечня (чтобы выбрать несколько,
зажмите клавишу Control/Command). Сам перечень можно редактировать, если перейти по ссылке под полем.

12. Дополнительные тэги, полезные метки, характеризующие какие-либо отдельные полезные составляющие отчетов.
Можно выбрать одно или несколько значений из перечня (чтобы выбрать несколько, зажмите клавишу Control/Command).
Сам перечень можно редактировать, если перейти по ссылке под полем.

После заполнения всех полей необходимо нажать кнопку "Добавить" внизу страницы!
