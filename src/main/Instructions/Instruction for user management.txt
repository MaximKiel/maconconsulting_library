Вход на сайт осуществляется по логину и паролю. Логин в нашем случае - первая часть рабочей почты сотрудника (до знака "@").
Например: для сотрудника с рабочей почтой "ivanov@mail.ru" логином будет являться "ivanov".
Пароль от сайта совпадает с паролем от рабочей почты.

Если система авторизации через логин и пароль не нужна, ее можно убрать - это опциональный функционал. Однако ее
рекомендуется оставить для безопасности. Логины и пароли также можно поменять (сделать более простыми при желании).
Для удобства рекомендуется включить в браузере функцию автозаполнения логина и пароля - это упростит вход на сайт.

Доступ к функционалу управления пользователями есть только у Администраторов (пользователи с ролью "Admin")!

Для добавления нового пользователя необходимо:
    1. Перейти на страницу "Управление пользователями (только admin)".
    2. Заполнить поля пользователя (имя, почту, пароль, роль). Роль выбирается из фиксированного списка: "User",
       "Manager", "Admin". В настоящий момент единственная разница между ролями пользователей - доступ к управлению
       пользователями есть столько у Admin. В остальном права доступа идентичны. Сами роли добавлены для того, чтобы
       в будущем, в случае необходимости, была возможность разграничить уровень доступа для разных пользователей
       (например, чтобы изменять, удалять и добавлять проекты могли только пользователи с ролью Manager или Admin).
    3. Нажать кнопку "Добавить".
    4. Внимание! Логин НЕ нужно никуда вводить при добавлении пользователя. Логин считывается автоматически исходя из
       введенной почты (для сотрудника с рабочей почтой "ivanov@mail.ru" логином будет являться "ivanov") и так же
       автоматически добавляется в БД.

Помимо добавления новых пользователей есть возможность изменять существующих пользователей. Изменять можно поля: имя,
логин, почта, роль.
Внимание! Пароль изменить нельзя. Единственный способ изменить пароль для входа - администратор удаляет пользователя
и добавляет его заново с измененным паролем. Причина: при регистрации пользователей пароль шифруется, поэтому нет
возможности "вытащить" из БД старый пароль для изменения. При необходимости можно прописать отдельный функционал,
чтобы можно было менять пароль без удаления пользователя. Это не делалось изначально по той причине, что сценарий
такого использования крайне редок и маловероятен в нашем случае (чаще всего пароль устанавливается при регистрации
пользователя и используется в дальнейшем без изменения).

Справа в шапке сайта есть кнопка "Выйти" - она производит Logout, то есть сайт "забывает" ваш логин и пароль, то есть
позволяет перезайти на сайт под другими логином и паролем.
